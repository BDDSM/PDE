Типы метрик:

Counter

Совокупная метрика, представляющая собой один монотонно увеличивающийся счетчик, значение которого может только увеличиваться или сбрасываться до нуля при перезапуске. Например, можно использовать счетчик для представления количества выполненных запросов, задач или ошибок.

Gauge

Метрика, представляющая одно числовое значение, которое может произвольно изменяться с течением времени как в большую, так и в меньшую сторону. Например, можно использовать для вывода данных APDEX.

Метод получения метрик:

По запросу (Pull)

Метрика расчитывается "на лету" при обращении (scrape) сервера Prometheus. Рекомендуется для легких, быстрых запросов.

Самостоятельно (Push)

Рекомендовано в случае, если есть потребность формировать метрику по результатам выполнения "тяжелых" вычислений, однако эти вычисления запускаются сторонними процессами. Требует установки шлюза Pushgateway на стороне Prometheus

Регламентом (условный Pull)

Рекомендовано в случае, если расчет метрики выполняется с помощью "тяжелых" вычислений, однако выполнение расчета не предполагается другими бизнес-процессами. Проще говоря - только в том случае, когда единственный потребитель метрики - это Prometheus
